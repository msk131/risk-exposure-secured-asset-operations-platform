# Glossary

This glossary expands common banking, architecture, cloud, and migration terms used across the workspace.

| Term | Full Form | Definition |
| --- | --- | --- |
| ADR | Architecture Decision Record | A short document that captures an important technical decision, why it was made, alternatives considered, and consequences. |
| API | Application Programming Interface | A contract that lets applications or services communicate through defined operations, requests, and responses. |
| API-first | Application Programming Interface-first | A design approach where service contracts are defined before implementation so channels and systems integrate predictably. |
| AWR | Automatic Workload Repository | Oracle performance repository that stores workload and database performance snapshots for tuning analysis. |
| AWS | Amazon Web Services | Amazon's public cloud platform used for compute, storage, networking, databases, security, and managed services. |
| BCBS 239 | Basel Committee on Banking Supervision standard 239 | Banking regulation guidance for risk data aggregation, accuracy, completeness, timeliness, and reporting governance. |
| CDC | Change Data Capture | A pattern that captures inserts, updates, and deletes from a source database and publishes them to another system or event stream. |
| CI/CD | Continuous Integration and Continuous Delivery | Automated build, test, security scan, package, and deployment workflow used to release software safely and repeatedly. |
| CBO | Cost-Based Optimizer | Oracle database component that chooses an execution plan based on estimated cost, table statistics, and indexes. |
| DLQ | Dead Letter Queue | A separate queue or topic used to isolate messages that cannot be processed successfully after defined retries. |
| DevSecOps | Development, Security, and Operations | Software delivery practice that integrates security checks, operational readiness, and automation into the development lifecycle. |
| EKS | Elastic Kubernetes Service | Amazon Web Services managed Kubernetes service for running containerized applications. |
| ERD | Entity Relationship Diagram | A diagram that shows business entities, fields, primary keys, foreign keys, and relationships between data objects. |
| FK | Foreign Key | A database constraint that links a field in one table to the primary key of another table. |
| GitOps | Git Operations | A deployment model where Git is the source of truth for infrastructure and application release state. |
| IAM | Identity and Access Management | A security capability for defining users, roles, permissions, policies, and service access. |
| IT | Information Technology | The discipline responsible for operating, securing, and supporting technology systems and infrastructure. |
| KMS | Key Management Service | Amazon Web Services service for creating, storing, rotating, and controlling encryption keys. |
| Kafka | Apache Kafka | Distributed event streaming platform used to publish, store, and consume high-volume messages. |
| ML | Machine Learning | A branch of Artificial Intelligence where systems learn patterns from data to make predictions or classifications. |
| MQ | Message Queue | Messaging middleware or pattern that lets systems exchange messages asynchronously. |
| MSK | Managed Streaming for Apache Kafka | Amazon Web Services managed Apache Kafka service. |
| mTLS | Mutual Transport Layer Security | A security pattern where both client and server authenticate each other using certificates. |
| NFR | Non-Functional Requirement | A system quality requirement such as availability, performance, security, auditability, scalability, or resiliency. |
| OpenTelemetry | Open Telemetry | Vendor-neutral standard for collecting metrics, logs, and distributed traces from applications. |
| PII | Personally Identifiable Information | Data that can identify a person, such as name, account number, tax identifier, phone number, or email address. |
| PL/SQL | Procedural Language extension to Structured Query Language | Oracle's procedural programming language used for stored procedures, functions, triggers, and packages. |
| RPO | Recovery Point Objective | Maximum acceptable amount of data loss measured in time after an incident. |
| RTO | Recovery Time Objective | Maximum acceptable time to restore service after an incident. |
| RBAC | Role-Based Access Control | Authorization model where permissions are assigned to roles, and users or services receive permissions through role membership. |
| S3 | Simple Storage Service | Amazon Web Services object storage service used for files, backups, archives, and data lakes. |
| SLA | Service Level Agreement | A formal service commitment, often contractual, for availability, response time, or support obligations. |
| SLO | Service Level Objective | A measurable reliability target, such as 99.9 percent availability or 200 millisecond p95 latency. |
| SOAP | Simple Object Access Protocol | XML-based web service protocol commonly used in older enterprise integrations. |
| SFTP | Secure File Transfer Protocol | Secure network protocol used to transfer files between systems. |
| SQL | Structured Query Language | Standard language used to query and modify relational databases. |
| SQL*Loader | Structured Query Language Loader | Oracle utility used to load data from external files into Oracle database tables. |
| VPC | Virtual Private Cloud | Isolated cloud network boundary used to control subnets, routing, security groups, and private connectivity. |
| W3C | World Wide Web Consortium | Standards organization responsible for web standards, including distributed trace context headers. |
| XML | Extensible Markup Language | Text-based markup format often used by older enterprise integrations and SOAP web services. |
