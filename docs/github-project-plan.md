# GitHub Project Plan

This file defines a suggested GitHub Projects setup for tracking the portfolio as a set of professional modernization workstreams.

## Project Name

Risk Exposure & Secured Asset Operations Platform

## Project Description

Portfolio of practical engineering plans for migration planning and new application design for a Risk Exposure & Secured Asset Operations Platform.

## Board Columns

| Column | Purpose |
| --- | --- |
| Backlog | Ideas, candidate work items, and future improvements |
| Discovery | Items being researched or scoped |
| Design | Architecture, schema, or migration design in progress |
| Build | Documentation, sample scripts, or examples being created |
| Validate | Review, quality checks, and consistency pass |
| Done | Completed and ready to present |

## Labels

| Label | Purpose |
| --- | --- |
| `migration-planning` | Legacy modernization and phased migration work |
| `oracle-performance` | Oracle tuning, SQL review, indexing, and batch optimization |
| `schema-design` | Data model, ownership, and schema evolution work |
| `migration-tooling` | Flyway, Liquibase, CI/CD, and automation |
| `observability` | Metrics, logs, tracing, dashboards, and readiness |
| `devsecops` | CI/CD, security controls, and release governance |
| `portfolio-polish` | README, positioning, writing, and presentation improvements |
