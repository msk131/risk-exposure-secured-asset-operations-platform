# Observability And DevSecOps

## Observability

| Signal | Purpose |
| --- | --- |
| Metrics | Service health, latency, error rate, throughput, queue lag |
| Logs | Request trace, business event context, exception details |
| Traces | Cross-service flow visibility |
| Dashboards | Operational and migration readiness monitoring |
| Alerts | Proactive incident detection |

## Key Dashboards

- API latency and error rate
- Event bus lag and failed messages
- Database connection and slow query trends
- Instruction workflow backlog
- Exposure calculation duration
- Asset allocation exceptions
- Audit event ingestion health
- Reporting projection lag

## DevSecOps Controls

- Static analysis and dependency scanning in CI.
- Contract tests for APIs and events.
- Database migration dry runs.
- Container image scanning.
- Secrets managed outside source control.
- Least-privilege service accounts.
- Environment-specific configuration.
- Deployment approvals for high-risk changes.
